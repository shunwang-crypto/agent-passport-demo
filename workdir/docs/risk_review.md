# 风险评审记录（发布前专项）

## 风险台账

1. 风险项：委托票据泄露导致重放
   - 等级：高
   - 触发信号：同一 capability 在短窗口内出现重复消费
   - 控制项：`max_uses=1` + 消费即写审计 + 回放 case 每日校验
   - 责任人：安全负责人（刘洋）

2. 风险项：任务上下文混淆导致跨任务误用
   - 等级：高
   - 触发信号：`task_id` 不一致却仍触发后续流程
   - 控制项：`task_id` 与 claims 双向匹配，不一致即拒绝
   - 责任人：网关负责人（李昊）

3. 风险项：高风险外发缺少审批票据
   - 等级：高
   - 触发信号：`approval_required=true` 且 `approval_ticket` 为空
   - 控制项：审批前置校验，缺票据直接 deny，并记录 `approval_missing`
   - 责任人：审批对接人（周宁）

4. 风险项：身份伪造或过期 token 访问
   - 等级：中
   - 触发信号：`auth_token` 无效、过期或不匹配 workload
   - 控制项：认证失败即终止，不进入授权决策
   - 责任人：身份平台（陈曦）

5. 风险项：审批服务抖动导致协作时效下降
   - 等级：中
   - 触发信号：审批回调 P95 > 2.5s
   - 控制项：启用降级策略（只允许低风险动作），并在群同步中显式标注未闭环项
   - 责任人：平台 SRE（王倩）

## 发布前必须完成的检查

1. 关键阻断场景（重放、篡改、缺审批）回放通过率 100%。
2. 审计日志抽样 20 条，字段完整率达到 100%。
3. 发生回滚时，15 分钟内可恢复至上一稳定版本。

## 评审结论

当前风险控制总体可接受，但审批抖动仍是主要不确定项。发布窗口内需要安排专人值守，并在 30 分钟节点复核一次关键指标。
