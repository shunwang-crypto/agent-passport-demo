# 四月发布说明（发布候选 RC2）

## 版本信息

- 版本号：v1.9.0-rc2
- 计划灰度时间窗：2026-04-19 20:00 至 22:00
- 影响范围：授权网关、委托签发、审计追踪、审批联动

## 本次发布目标

1. 把授权拒绝原因结构化到可审计、可追踪、可回放。
2. 将高风险外发纳入审批闭环，避免“漏票据放行”。
3. 提升跨任务票据误用拦截能力，降低重放与混淆风险。

## 变更清单

1. 新增 `task_id` 双向绑定校验，阻断跨任务复用 capability。
2. 调整签名校验顺序，篡改票据在前置阶段直接失败并返回标准错误码。
3. 审计日志新增链路字段：`policy_rule`、`delegation_id`、`review_reason_code`。
4. 修复 `max_uses=1` 在并发竞争下的边界条件，避免重复消费漏拦截。
5. 审批模式新增 preview/full 区分，便于联调阶段压低噪声。

## 发布门槛（全部满足才可放量）

- 11 个基准场景全部符合预期
- 阻断类场景拦截率保持 100%
- 授权决策 P95 < 50ms
- 审批回调 P95 < 2.5s
- 灰度 30 分钟内无 P1 告警

## 灰度与回滚策略

1. 第一阶段（20:00-20:30）仅开启 10% 低风险流量。
2. 第二阶段（20:30-21:00）提升到 40%，重点观察审批延迟与拒绝率。
3. 若出现以下任一条件立即回滚：
	- 授权错误率 > 0.5%
	- 审批回调超时率 > 2%
	- 审计写入失败连续 5 分钟
4. 回滚后由平台负责人在 15 分钟内提交复盘与二次上线建议。
