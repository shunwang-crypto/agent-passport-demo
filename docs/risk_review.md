# 风险评审纪要

## 风险项与控制项

1. 风险项：委托令牌泄露后存在重放风险。
   控制项：max_uses=1，访问后立即消费并记录审计事件。

2. 风险项：任务上下文混淆导致跨任务复用。
   控制项：task_id 与 capability claims 双向匹配，不一致即拒绝。

3. 风险项：审批票据缺失导致高风险外发越权。
   控制项：approval_required 为真时必须校验 approval_ticket。

4. 风险项：身份伪造导致冒名调用。
   控制项：工作负载身份认证失败时直接拒绝，不进入后续授权路径。

5. 风险项：签名篡改导致伪造能力票据。
   控制项：签名校验失败前置拦截，并记录 capability_invalid_signature。

## 结论

当前控制链路可以覆盖主要协作风险，建议持续进行异常注入与回放验证。
